# 飞机大战局域网连接使用指南

## 功能简介

局域网连接功能允许运行在不同主机上的Android模拟器客户端连接到局域网内任意主机上的游戏服务器进行对战。

## 快速开始

### 第一步：启动游戏服务器

1. 在你的电脑上启动 AirWarServer 服务器
2. 服务器启动后会在控制台显示监听地址，例如：
   ```
   Server is listening on: 0.0.0.0:18080
   Access from LAN using: 192.168.1.100:18080
   ```
3. 记下局域网IP地址（如 `192.168.1.100`）

### 第二步：配置Android客户端

#### 方法一：自动扫描（推荐）

1. 在Android客户端主界面，点击**设置**按钮
2. 在设置界面，点击**"Scan LAN Servers"**（扫描局域网服务器）按钮
3. 等待扫描完成（通常需要10-30秒）
4. 扫描结果会显示发现的服务器列表，包含IP地址和响应时间
5. 点击列表中的服务器，IP地址和端口会自动填入配置框
6. 点击**"Test Connection"**（测试连接）验证连接
7. 如果测试成功，点击**"Save"**（保存）按钮保存配置

#### 方法二：手动配置

1. 在Android客户端主界面，点击**设置**按钮
2. 在"Server IP Address"（服务器IP地址）框中输入服务器的局域网IP
   - 例如：`192.168.1.100`
3. 在"Server Port"（服务器端口）框中输入端口号
   - 默认端口：`18080`
4. 点击**"Test Connection"**（测试连接）按钮
5. 如果显示绿色成功提示，点击**"Save"**（保存）按钮
6. 配置会自动保存，下次启动应用时无需重新配置

### 第三步：开始游戏

配置保存后，客户端会使用新的服务器地址进行所有网络连接。你现在可以：
- 登录账号
- 匹配对战
- 查看排行榜
- 使用所有在线功能

## 常见场景

### 场景1：本机测试（模拟器连接本机服务器）

**服务器IP地址：** `10.0.2.2`  
**服务器端口：** `18080`

这是Android模拟器访问宿主机的特殊地址，适用于服务器和模拟器在同一台电脑上运行的情况。

### 场景2：局域网对战（不同电脑）

**服务器IP地址：** `192.168.x.x`（服务器主机的局域网IP）  
**服务器端口：** `18080`

适用于服务器和客户端在不同电脑上运行，但在同一局域网内的情况。

## 如何查找服务器IP地址

### Windows系统

1. 按 `Win + R` 打开运行对话框
2. 输入 `cmd` 并回车打开命令提示符
3. 输入命令：`ipconfig`
4. 查找"以太网适配器"或"无线局域网适配器"下的"IPv4 地址"
5. 通常是 `192.168.x.x` 或 `10.0.x.x` 格式

### Mac/Linux系统

1. 打开终端
2. 输入命令：`ifconfig`（Mac）或 `ip addr`（Linux）
3. 查找 `inet` 后面的IP地址
4. 通常是 `192.168.x.x` 或 `10.0.x.x` 格式

## 故障排查

### 问题1：测试连接失败，提示"无法连接到服务器"

**可能原因：**
- 服务器未启动
- IP地址输入错误
- 客户端和服务器不在同一局域网
- 防火墙阻止了连接

**解决方法：**
1. 确认服务器已启动并显示监听地址
2. 检查IP地址是否正确（可以使用扫描功能自动发现）
3. 确保两台设备连接到同一个WiFi或局域网
4. 临时关闭防火墙测试，或添加端口18080的防火墙规则

### 问题2：扫描找不到服务器

**可能原因：**
- 服务器未启动
- 服务器和客户端不在同一个C类子网（如192.168.1.x）
- 网络响应超时

**解决方法：**
1. 确认服务器已启动
2. 检查服务器和客户端的IP地址是否在同一网段
3. 使用手动配置方式直接输入服务器IP

### 问题3：保存后仍然连接失败

**可能原因：**
- 配置保存后需要重新建立连接
- 旧的WebSocket连接未断开

**解决方法：**
1. 完全退出应用并重新启动
2. 重新测试连接确认配置正确

### 问题4：端口错误提示

**可能原因：**
- 端口号输入错误
- 服务器使用了不同的端口

**解决方法：**
1. 确认服务器启动日志中显示的端口号
2. 确保端口号在1-65535范围内
3. 默认端口是18080

## 技术说明

### 服务器配置

服务器已配置为监听所有网络接口（0.0.0.0），这意味着：
- 可以接受来自任何IP地址的连接
- 支持本机连接（localhost/127.0.0.1）
- 支持局域网连接（192.168.x.x）
- 向后兼容，不影响现有功能

### 客户端配置

客户端使用动态服务器配置系统：
- 配置持久化存储在SharedPreferences中
- 修改配置后立即生效，无需重启应用
- 支持实时连接测试
- 自动断开旧连接并使用新地址

### 网络要求

- 服务器和客户端必须在同一局域网内
- 防火墙需要允许端口18080的TCP连接
- 建议使用稳定的WiFi或有线网络连接

## 安全提示

1. 局域网连接仅在可信网络环境中使用
2. 不要在公共WiFi网络中使用此功能
3. 确保服务器运行在安全的网络环境中
4. 定期更新服务器和客户端到最新版本

## 常见问题（FAQ）

**Q: 配置会在应用重启后保留吗？**  
A: 是的，配置会自动保存并在下次启动时加载。

**Q: 可以同时连接多个服务器吗？**  
A: 不可以，客户端同一时间只能连接一个服务器。

**Q: 扫描功能会扫描整个网络吗？**  
A: 不会，扫描仅限于当前设备所在的C类子网（如192.168.1.0/24），最多扫描254个IP地址。

**Q: 为什么扫描需要这么长时间？**  
A: 扫描需要逐个测试子网中的每个IP地址，每个IP的超时时间为500毫秒，因此完整扫描可能需要10-30秒。

**Q: 可以修改服务器端口吗？**  
A: 可以，但需要同时修改服务器配置文件（application.yml）和客户端设置中的端口号。
