# 模板模式设计说明（游戏难度）

## 类图

参见 `GameModeTemplate.puml`。

## 角色说明

- **AbstractGameMode（抽象模板类）**：定义 `apply()`、`update()`、`onBossSpawn()` 三个模板方法，固定初始化流程（设置敌机上限、生成周期、Boss 配置等）并留出若干抽象步骤供子类实现。内部通过钩子方法配置敌机工厂、Boss 工厂以及难度提升逻辑。
- **EasyGameMode / NormalGameMode / HardGameMode（具体模板子类）**：分别实现钩子方法，提供不同的初始参数与动态难度调整策略。普通、困难模式会随时间输出难度提升日志并调节敌机数量、生成周期、精英概率等；困难模式额外提升 Boss 血量与刷新频率。
- **Game（客户端 / 上下文）**：在构造函数中选择具体模式并调用 `apply()` 完成配置，在游戏循环中调用 `update()`，在 Boss 生成后调用 `onBossSpawn()` 以便模式自定义后续行为。

通过模板模式，三种难度共享统一的初始化和更新流程，只需在子类中覆写关键步骤即可扩展或调整难度逻辑。
