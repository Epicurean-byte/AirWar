# 飞机大战 Android 项目 - 整体设计文档

## 项目概述

本项目是一个基于 Android 平台的飞机大战游戏，支持单机模式和联机对战/合作模式。项目采用客户端-服务器架构，实现了完整的游戏逻辑、用户系统、商城系统和排行榜功能。

### 2. 模块划分

#### 2.1 Android 客户端 (android-client/)
- **UI 模块**: 用户界面，包括主菜单、游戏界面、商城、排行榜等
- **游戏核心模块**: 游戏逻辑、渲染、碰撞检测等
- **网络模块**: HTTP API 调用和 WebSocket 实时通信
- **数据持久化模块**: SQLite 本地数据库，存储游戏记录和用户数据

#### 2.2 服务器端 (AirWarServer/)
- **用户管理**: 用户注册、登录、好友系统
- **游戏服务**: 房间管理、匹配系统、游戏状态同步
- **商城系统**: 皮肤购买、金币管理
- **排行榜**: 分数统计和排名

#### 2.3 游戏核心库 (game-core/)
- 共享的游戏逻辑代码
- 实体类定义
- 工具类

## 核心功能

### 1. 单机模式
- 经典的飞机大战玩法
- 敌机生成和 AI
- 道具系统
- 分数统计

### 2. 联机模式
- **对战模式 (PVP)**: 两个玩家相向而行，各自在屏幕的上下半部分
- **合作模式 (COOP)**: 两个玩家并排作战，共同对抗敌机
- 实时同步游戏状态
- 子弹轨迹渲染
- 皮肤同步

### 3. 用户系统
- 用户注册和登录
- 好友系统
- 用户资料管理

### 4. 商城系统
- 飞机皮肤购买
- 金币系统
- 库存管理

### 5. 排行榜
- 全局排行榜
- 好友排行榜
- 实时更新

### 6. 局域网发现
- 自动扫描局域网内的服务器
- 一键连接
- 服务器配置管理

## 数据流

### 游戏数据流
```
1. 单机模式:
   用户输入 → 游戏逻辑 → 渲染 → 结算 → 本地数据库

2. 联机模式:
   用户输入 → WebSocket → 服务器 → 游戏逻辑 → 
   广播状态 → WebSocket → 客户端渲染 → 结算 → 
   HTTP API → 服务器 → 本地数据库
```

### 用户数据流
```
注册/登录 → HTTP API → 服务器验证 → 返回用户信息 → 
本地缓存 → 本地数据库
```

## 性能优化

### 客户端
1. **渲染优化**: 使用 SurfaceView 进行游戏渲染
2. **内存管理**: Bitmap 复用，及时释放资源
3. **网络优化**: 减少不必要的网络请求，批量发送数据
4. **数据库优化**: 使用索引，批量操作

### 服务器端
1. **并发处理**: 使用线程池处理游戏逻辑
2. **状态同步**: 固定频率广播（50ms/tick）
3. **内存管理**: ConcurrentHashMap 管理游戏房间

## 安全性

1. **用户认证**: 基于 userId 的简单认证
2. **数据验证**: 服务器端验证所有输入
3. **WebSocket 安全**: 连接前必须认证
4. **本地数据加密**: 敏感数据加密存储（待实现）

## 部署

### 服务器部署
```bash
cd AirWarServer
mvn clean package
java -jar target/planewar-server.jar
```

### 客户端部署
```bash
cd android-client
./gradlew assembleDebug
# APK 位于 app/build/outputs/apk/debug/
```
