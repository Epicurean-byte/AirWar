## 核心技术实现

### 1. 实时游戏同步

**技术方案**: WebSocket + 服务器权威

```
客户端输入 → WebSocket → 服务器处理 → 
广播状态 → 客户端渲染
```

**同步频率**: 20 TPS (每50ms一次)

**数据结构**:
```json
{
  "tick": 123,
  "players": [...],
  "enemies": [...],
  "bullets": [...]
}
```

### 2. 游戏模式设计

#### PVP 模式
- 玩家1在屏幕上半部分
- 玩家2在屏幕下半部分
- 各自击杀敌机获得分数
- 游戏结束比较分数

#### COOP 模式
- 两个玩家并排在屏幕下方
- 共同对抗敌机
- 分数独立计算
- 任一玩家死亡游戏结束

### 3. 本地数据库设计

#### 数据表结构

**游戏记录表** (game_records):
- 存储每局游戏的详细数据
- 包括分数、金币、时长、击杀数等
- 支持按模式、时间查询

**用户统计表** (user_stats):
- 累计统计数据
- 总场次、总分数、最高分
- PVP 胜率、COOP 场次

#### 数据库特性
- 单例模式管理
- 自动更新统计
- 索引优化查询
- 支持数据迁移

## 遇到的问题和解决方案

### 1. 子弹轨迹不显示
**问题**: 联机模式下子弹轨迹不可见

**原因**: 
- 子弹创建和渲染逻辑正确
- 可能是 Bitmap 加载问题

**解决方案**:
- 添加详细日志
- 验证 Bitmap 加载
- 检查渲染顺序

### 2. 模式切换按钮无响应
**问题**: UI 按钮点击后没有视觉反馈

**解决方案**:
- 使用背景颜色变化
- 绿色表示合作模式选中
- 橙红色表示对战模式选中
- 灰色表示未选中

### 3. 游戏状态同步延迟
**问题**: 多人游戏时偶尔出现卡顿

**解决方案**:
- 优化网络消息频率
- 减少不必要的数据传输
- 使用对象池减少 GC
